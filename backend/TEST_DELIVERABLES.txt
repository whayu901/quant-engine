1. Install Test Dependencies:
   $ cd backend
   $ pip install -r tests/requirements-test.txt

2. Run All Tests:
   $ pytest tests/

3. Run with Coverage:
   $ pytest tests/ --cov=app --cov-report=html

4. Run Specific Test Type:
   $ pytest tests/ -m unit              # Unit tests only
   $ pytest tests/ -m integration       # Database tests
   $ pytest tests/ -m api               # API tests

5. Use Test Runner Script:
   $ ./run_tests.sh                     # All tests with coverage
   $ ./run_tests.sh --fast              # Quick run

SUPPORTED TEST COMMANDS
================================================================================

# Run all tests
pytest tests/

# Run with coverage
pytest tests/ --cov=app --cov-report=html

# Run specific test type
pytest tests/ -m unit              # Unit tests
pytest tests/ -m integration       # Integration tests
pytest tests/ -m api               # API tests

# Run by marker
pytest tests/ -m auth              # Auth tests
pytest tests/ -m db                # Database tests
pytest tests/ -m "not slow"        # Exclude slow tests

# Run specific file
pytest tests/test_unit_security.py

# Run specific test class
pytest tests/test_unit_security.py::TestPasswordHashing

# Run specific test
pytest tests/test_unit_security.py::TestPasswordHashing::test_hash_password

# Run in parallel (faster)
pytest tests/ -n 4                 # 4 workers
pytest tests/ -n auto              # Auto-detect cores

# Verbose output
pytest tests/ -v
pytest tests/ -v -s                # Include print statements

# Show slowest tests
pytest tests/ --durations=10

# Use test runner script
./run_tests.sh
./run_tests.sh --help

================================================================================
TROUBLESHOOTING
================================================================================

Problem: ModuleNotFoundError: No module named 'app'
Solution: conftest.py adds parent directory to sys.path

Problem: Database is locked
Solution: Using in-memory SQLite with automatic cleanup

Problem: Token verification failed
Solution: Ensure test fixtures set user._password attribute

Problem: Tests fail individually but pass together
Solution: Check for test interdependencies (shouldn't exist)

Problem: Coverage report missing files
Solution: Check pytest.ini [coverage:run] section

See TEST_IMPLEMENTATION_GUIDE.md for more troubleshooting.
