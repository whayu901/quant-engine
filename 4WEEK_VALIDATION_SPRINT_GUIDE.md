# QUAL ENGINE - 4-WEEK VALIDATION SPRINT GUIDE
**Practical Research Framework for Critical Assumption Testing**

**Duration:** 28 days (4 weeks)
**Investment:** $2,000-3,000
**Resource Time:** 60-80 hours
**Decision Point:** Day 21 (GO/NO-GO by July 16, 2026)
**Launch Readiness:** Day 28

---

## SPRINT OVERVIEW

### Week 1: Research & Interview (Days 1-7)
- Interview 10-15 SEA researchers
- Deep competitive landscape scan
- Preliminary pricing sensitivity test

### Day 3-4: Interview Design & Preparation

#### Interview Guide Template (30 minutes)

**Setup (1 minute):**
```
Hi [Name], thanks so much for making time. I'm researching how qual researchers
in Southeast Asia work and what tools they use. This is a research conversation,
not a sales pitch. Your honest feedback is most helpful to us. Do you have any
questions before we start?

[Record audio? "I'd like to record this so I can focus on our conversation
rather than taking notes - is that OK?"]
```

**Opening Question (3 minutes):**
```
1. "Walk me through your last qualitative research project from start to finish.
   What was it? How did it go?"

[Goal: Get them talking about real workflow, not hypothetical]
[Listen for: Tools mentioned, pain points expressed, time spent]
```

**Pain Point Exploration (8 minutes):**
```
2. "What's the most time-consuming part of that process?"
   [Follow-up if needed: "How much time? Can you give an example?"]

3. "What's the most frustrating part of your job as a researcher?"
   [Goal: Identify #1 pain, not suggested pains]

4. "How do you currently handle interviews/data in languages other than English?"
   [If they don't mention language:]
   "Do you ever work with Indonesian/Thai/mixed-language transcripts?"
   [If yes:] "How do you handle those?"

[Goal: Understand current language workarounds, not theoretical pain]
[Listen for: Do they name language? Rank it vs other pains?]
```

**Tools & Solutions (6 minutes):**
```
5. "What tools are you currently using for qual research?"
   [For each tool:] "What do you like about it? What's missing?"

6. "Have you looked at any other tools? Why'd you choose your current solution?"
   [Goal: Understand switching costs, competitive landscape]

7. "If you could redesign your qual research workflow, what would you change?"
   [Goal: Unprompted feature requests, ideation]
   [Don't suggest features - let them ideate]

[Listen for: Do they mention code-mixing/language? How much weight does it get?]
```

**Solution Testing (8 minutes - use ONLY if they've validated pain):**
```
8. "Imagine a tool that could analyze interviews in Indonesian/Thai/mixed
   languages AND detect when people are code-mixing. How valuable would that be
   to you? On a scale of 1-10?"

   [If 7+: Continue to 9]
   [If <7: Skip to 10]

9. "How much would you be willing to pay per month for that feature?"
   [If hesitant: "Is it more than $5? More than $10? More than $25?"]

10. "What would it take for you to switch from [current tool] to a new platform?"
    [Goal: Understand switching costs, required features]
    [Listen for: Price? Features? Integration? Trust/brand?]
```

**Closing (2 minutes):**
```
11. "Is there anything else I should know about qual research in Southeast Asia?"

12. "Would you be open to testing a new tool once we build it? I'll follow up
    in a few months with early access."
    [Goal: Capture for beta cohort]

[Thank them, confirm incentive delivery (email link or in-app credit)]
```

### Day 9-10: Process Transcripts & Generate Analyses

**For each of the 5 transcripts:**

**Step 1: Transcription (If needed)**
- If audio/video: Use Deepgram or manual transcription
- If text: Skip this step
- Timeline: 1-2 hours

**Step 2: Theme Extraction (Using existing backend)**
- Run your mock Claude API (theme extraction)
- Generate 5-7 key themes
- Create evidence grid (theme × speaker)
- Collect 3-5 verbatims per theme

**Step 3: Code-Mixing Detection**
- Flag if code-mixing detected
- Highlight mixed-language quotes
- Note language switch points

**Step 4: Format for User Review**
- Excel spreadsheet with:
  - Themes (Column A)
  - Number of mentions (Column B)
  - Key verbatims (Column C)
  - Speaker names (Column D)
  - Timestamp (Column E)

- OR: Google Slides presentation with:
  - Slide 1: Project overview
  - Slide 2-3: Key themes (2 per slide)
  - Slide 4: Top insights
  - Slide 5: Participant summary

**Output:**
- 1 Excel file + 1 PDF slide deck per transcript
- Non-branded (no "Qual Engine" logo - this is manual service, not software)

### Day 19: Decision Framework

**Score interpretation:**

```
75-100 points: ✅ FULL GO
  - Core assumptions validated
  - Market shows strong signals
  - Proceed with full frontend development
  - No major pivots needed

50-74 points: 🟡 CONDITIONAL GO
  - Some assumptions validated, some weak
  - Can proceed but with revisions:
    * Adjust pricing (if criterion 2 weak)
    * Add hybrid mode for AI (if criterion 3 weak)
    * Niche positioning (if criterion 5 weak)
    * Emphasize strongest differentiator (if criterion 4 weak)
  - Proceed with REVISED plan

25-49 points: 🔴 PIVOT REQUIRED
  - Multiple assumptions invalidated
  - Can't proceed as planned
  - Options:
    * PIVOT A: Change positioning (e.g., "speed" not "language")
    * PIVOT B: Change segment (e.g., "agencies" not "in-house")
    * PIVOT C: Change business model (e.g., "services" not "software")
  - Spend 1 week redesigning, then re-validate

0-24 points: ⛔ STOP
  - Core assumptions rejected
  - Market doesn't support planned product
  - Decision: Exit, find new market, or radical pivot
  - Schedule founder conversation before proceeding
```

### Day 20-21: Decision & Planning

**If FULL GO (75+):**
```
1. Celebrate! Core assumptions validated.
2. Proceed with frontend development as planned
3. Timeline: Launch Month 1-2 (on schedule)
4. Targets: 40 customers, $4K MRR by Month 1
5. Focus: Activate quickly, measure retention
```

**If CONDITIONAL GO (50-74):**
```
1. Identify specific revisions needed
2. Revise business plan (pricing, features, positioning, targets)
3. Share revised plan with team & advisors
4. Proceed with frontend AFTER revisions approved
5. Timeline: Launch Month 2-3 (1-month delay acceptable)
6. Targets: Revise to realistic numbers (e.g., 100 customers by Month 6, not 500)
```

**If PIVOT REQUIRED (25-49):**
```
1. Schedule 2-hour founder meeting
2. Analyze: Which core assumption failed?
3. Design pivot:
   - PIVOT A: Keep product, change positioning/go-to-market
   - PIVOT B: Keep market, change product features
   - PIVOT C: Keep technology, change entire business model
4. Plan: 1-week re-design sprint
5. Re-validate with 5-10 new interviews
6. Only then proceed with development
```

### Appendix C: Pricing Sensitivity Calculator

**Van Westendorp Price Sensitivity Analysis:**

```
Ask each respondent 4 questions:

1. "At what price would you consider [product] too cheap (low quality)?"
   → Price Too Cheap: $___

2. "At what price would you consider [product] a good value for money?"
   → Price Acceptable: $___

3. "At what price would you consider [product] getting expensive?"
   → Price Getting Expensive: $___

4. "At what price would you consider [product] too expensive (wouldn't buy)?"
   → Price Too Expensive: $___

ANALYSIS:
- Optimal price = Intersection of "good value" and "getting expensive"
- Range = "Good value" to "Getting expensive"
- Plot responses to visualize price zone

Example output (from hypothetical 20-person study):
- Optimal price: $49-69/month
- Too low: <$20/month
- Too high: >$150/month
```
