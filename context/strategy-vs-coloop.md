# Strategy vs CoLoop.ai

## Current state (cloned repo, 2026-06-30)
The repo has **already built a CoLoop-parity qual platform** (analysis_grid, concept_testing,
content_analysis, open_ends, rag/chat, clips, live_recorder, integrations — Phase 0-7). The
**Fieldwork QC wedge does not exist in code** (grep: nothing). So today it's a strong but **me-too**
platform with no moat. **Direction A** corrects this: parity = table-stakes; build the moat next.

## What CoLoop already owns (do not fight here now)
YC-backed, international, qual-native: transcription (40+ langs), analysis grids, RAG repository,
open-end coding, concept testing, clips, evidence/citations, toplines, reusable "Skills",
SOC2/GDPR/residency + integrations. **We will not out-CoLoop CoLoop on analysis** — and we don't
need to, because we already have a parity base.

## Where we win — layers CoLoop structurally cannot reach
1. **Fieldwork / data-collection integrity (THE WEDGE).** CoLoop starts at the transcript — it never
   touches *how data was collected*. Verifying interviews happened, anti-curbstoning, silent-audio /
   registrant verification at scale. Validated by real Kadence work (600+ files verified/day,
   MyPertamina). No incumbent sells this to them. **Build this next.**
2. **Southeast Asia layer.** Code-mix languages (Taglish/Singlish/Manglish/Bahasa+English), in-region
   hosting + UU PDP/PDPA, local integrations (WhatsApp, SurveyToGo/Dooblo, tSurvey, KOINS), local
   currency & pricing. The reason to exist vs a global tool.
3. **Quant ops.** Kadence is quant-heavy; CoLoop/most qual tools don't do crosstabs / quota /
   code-frame / tabulation. Open ground — a later layer.

## The one-liner
> We don't beat CoLoop at its game. We win on the layers it can't reach:
> **collection integrity + SEA + quant ops.** Parity is just the floor.

## Order of work
1. Fieldwork QC wedge -> 2. SEA layer real -> 3. (later) broader CoLoop competition + quant depth.
